# 系统分析报告

## 1. 总体评估
当前系统 CPU 负载极低，所有活跃进程的 CPU 占用率均低于 **0.5%**，无进程触及异常阈值。
系统整体处于**空闲状态**，调度压力小，不存在 CPU 争抢或资源饥饿现象，运行健康状况良好。

## 3. 异常根因判断
**本期采集未发现任何异常进程** （异常进程数 = 0）。
所有进程 CPU 占用远低于阈值（50%/90%），且上下文切换、调度延迟、迁移次数等指标均处于健康范围，因此不需要进行根因分类。

## 4. 关联分析
- **上下文切换**：绝大多数切换属于自愿切换（voluntary），表明线程主要在等待 I/O 或 timer 事件，而非 CPU 抢占。被动切换几乎为 0，说明 CPU 竞争极低。
- **调度延迟**：最大调度延迟出现在内核线程 `kworker/1:1-eve`（max **4187.8μs**，平均 **20.9μs**），但其 CPU 占用仅 0.1%，且仅有自愿切换。该延迟属于内核工作队列的瞬时调度抖动，未引起任何连锁影响，且普通用户态线程延迟均在个位数微秒，完全正常。
- **futex 等待与锁竞争**：所有进程的 futex 等待次数均为 **0**，表明无用户态锁竞争。
- **核间迁移**：迁移次数极低（0–3次/5秒），进一步证实 CPU 调度压力很小。

## 5. 优化建议
1. **无需任何性能调优操作**。当前系统 CPU 空闲率高，进程行为健康，建议保持现有配置。
2. **若需更精确的 CPU 记账**，可考虑启用 `schedstats` （当前采集依赖挂墙时间，可能因抢占被高估/低估），但当前负载下影响可忽略。
3. **内核线程 `kworker/1:1-eve` 的 millisecond 级延迟**非持续性问题，一般由设备事件处理引起，仅当其出现持续高延迟或高 CPU 时，再检查设备驱动或中断绑定。

## 6. 风险提示
- **CPU 计时精度**：因 `schedstats` 未启用，CPU% 基于挂墙时间采样，非精确的调度实体时间。不过在当前极低负载下，计算误差不会掩盖实质性问题。
- **采集粒度**：5 秒窗口可能遗漏瞬时 CPU 尖峰，但连续采集未发现异常进程，说明系统稳定性良好。
- **符号解析**：无缺失。所有进程名均在用户态呈现，无需人工进一步确认。
